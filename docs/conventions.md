# Convenção de Nomes de Variáveis

## Objetivo
Padronizar a nomenclatura de variáveis para melhorar legibilidade, manutenção e colaboração no projeto.

## Regras Gerais
- Utilizar nomes descritivos e significativos.
- Utilizar inglês para identificadores.
- Evitar abreviações ambíguas.
- Não utilizar acentos ou caracteres especiais.

## Padrões
### Variáveis
`camelCase`

### Constantes
`UPPER_SNAKE_CASE`

### Booleanos
Prefixos: `is`, `has`, `can`, `should`.

### Arrays
Nomes no plural.

### Objetos
Nomes no singular.

## Exemplos
```js
const MAX_RETRY_COUNT = 3;
let userName;
let isActive;
let users;
let user;
```
